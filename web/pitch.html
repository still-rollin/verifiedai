THE PROBLEM

A trillion-dollar trust story rests on machine-checked proofs of human-unchecked statements

Lean's kernel guarantees a proof matches its statement — and nothing more. If the statement is vacuous, simplified, or subtly wrong, you get a machine-certified proof of nothing, and no alarm ever rings.

>50%of miniF2F's failing problems were statement bugs — the field's flagship benchmark, wrong for 5 years (arXiv:2511.03108)

THE PRODUCT

CI for formal proofs — Dependabot + Codecov for Lean

STATEMENT AUDIT

Every check is a real Lean compilation: derive False from the hypotheses (vacuity), probe for trivially-true statements, surface unused hypotheses. No heuristics.

PROOF REPAIR

mathlib breaks downstream proofs every release. Repair ladder: library search → tactic cascade → LLM — and only kernel-verified fixes are kept. It cannot ship a wrong fix.

MCP-NATIVE

AI agents write proofs at industrial scale; verifiedai is the MCP tool that audits what they prove, from inside Claude Code, Cursor, or any agent loop.

Working today: CLI + GitHub Action + MCP server, live at github.com/still-rollin/verifiedai.
