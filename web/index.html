Maintained corpora are clean. Static and auto-generated ones are not, and nothing re-audits them. A defective statement in a benchmark skews a score; in training data it teaches a prover to exploit contradictions. Both are CI problems.

⊢Two commands

audit flags theorems the kernel accepts but that say nothing: contradictory hypotheses, conclusions that hold with every hypothesis stripped, unused hypotheses, smuggled axioms, hidden sorry, native_decide. repair fixes broken proofs — library search, tactic cascade, then an LLM — and applies a fix only if the file recompiles under the kernel.

$ verifiedai audit ./project Demo/Pricing.lean

  ✓ discount_le_price (line 10)
  ✗ big_discount_still_positive (line 16)
      [vacuous] the hypotheses are contradictory — False is derivable
      from them alone; this theorem proves nothing about any real case
  ⚠ tax_identity (line 22)
      [trivial] the conclusion holds without the hypotheses
      [unused_hypothesis] hypothesis `h` is never used

  3 theorems audited, 2 flagged

$ verifiedai repair ./project

  ✓ fixed Demo/Refunds.lean theorem refund_comm
      via exact? → exact Nat.add_comm a b
  1/1 broken declarations repaired (kernel-verified)

§Not just mathematics

The engine is domain-agnostic. We compiled Indian income-tax slabs, GST thresholds and TDS sections into Lean from their rule tables, proved the boundary cases in the kernel, planted two realistic mis-formalizations, and caught both — one "exemption rule" whose mixed-up thresholds exempt nobody, one theorem that compares constants instead of a business's turnover. Rules with money attached fail exactly like benchmark statements: the compliance pilot.

∀Who pays

Labs training provers. A model optimizing "the kernel accepted it" learns to smuggle axioms and mine contradictions. Its training data already contains both. verifiedai gates the reward signal and audits the corpus before the run.

Benchmark and dataset maintainers. Fixing miniF2F's statements moved measured prover accuracy from 40% to 70% (arXiv:2511.03108). A sweep is one command, about an hour per thousand statements.

Lean project maintainers. mathlib bumps break downstream proofs; PRs introduce vacuous lemmas reviewers wave through. Repair on bump, audit on PR, in CI.
