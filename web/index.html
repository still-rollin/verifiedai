CI FOR FORMAL PROOFS · LEAN 4

The kernel checks your proofs.
Nobody checks your statements.

A formally verified theorem is only as true as its statement. verifiedai is a statement auditor and proof-repair bot for Lean: every finding is a compiled Lean probe — no heuristics, no false positives — and every repair is accepted only if the kernel verifies it.

The gradient is the point: maintained corpora are clean; static and auto-generated ones are not — and nothing in the ecosystem re-audits the static ones. A defective statement in a benchmark skews a score. The same statement in training data is a live reward signal teaching a prover to exploit contradictions. That is a CI problem, and CI is the product.

What it does

verifiedai audit flags theorems the kernel happily accepts but that don't say what you think: contradictory (vacuous) hypotheses, statements that close under one tactic with all hypotheses stripped, unused hypotheses, and trust-chain cheats — smuggled axioms, hidden sorry, native_decide.

verifiedai repair fixes proofs broken by dependency bumps with a deterministic-first ladder — library search (exact?), tactic cascade, then an LLM with compile-error feedback. A candidate fix ships only if the file recompiles under the kernel. A wrong fix cannot ship, by construction.

$ verifiedai audit ./project Demo/Pricing.lean

  ⚠ tax_identity (line 22)

How the audit works

The auditor turns each theorem into adversarial probe programs and lets the Lean kernel judge them. A probe that compiles is a mathematical fact about your statement, not an opinion.

1Parse the claim

Split every theorem into hypotheses and conclusion.

theorem big_discount_still_positive
  (p d : Nat) (h1 : d > p + 10)
  (h2 : d < p) :          -- hypotheses
  applyDiscount p d > 0   -- conclusion

2Attack it

The vacuity probe keeps the hypotheses and swaps the conclusion for False; the triviality probe keeps the conclusion and strips the hypotheses.

-- if this COMPILES, the theorem
-- is about nothing
theorem __vac (p d : Nat)
  (h1 : d > p + 10) (h2 : d < p) :
  False := by
  first | omega | simp_all | decide

3Let the kernel judge

Probes are batched (≤20 theorems per compile) and inserted inside each theorem's own namespace, so auditing costs ~2 compiles per file. Each probe carries a #print axioms sentinel: a finding requires positive confirmation, so if Lean ever aborts mid-file the engine fails safe rather than misreading silence.

The checks

checkwhat it catches
vacuous contradictory hypotheses — the theorem is true about nothing LIVE
trivial conclusion closes under rfl / omega / decide / simp with all hypotheses stripped — no mathematical content survived formalization LIVE
unused hypothesis assumptions the declaration never uses — over-constrained or mis-formalized LIVE
trust chain the cheats an RL-trained prover learns: smuggled axioms, sorry hidden behind conjunctions or macros, native_decide — read from Lean's own axiom introspection. This is what makes verifiedai a reward-integrity gate for prover training LIVE
faithfulness an LLM that never saw the informal statement back-translates the formal one; a separate judge compares (--faithful, BYO key). Soft signal — warnings only BETA
definition drift redefine Prime locally, prove something easy about it — statements bound to shadowed definitions ROADMAP
spec mutation for verified code: inject known-buggy implementations; if the spec still verifies, the spec is too weak ROADMAP
Rocq/Coq support the same battery for the Rocq ecosystem ROADMAP

A finding that fires is a proven fact — the probe compiled. False negatives are expected: the battery is a lower bound, and every check added widens it. The adversarial zoo — a corpus of kernel cheats the auditor must catch forever, enforced in CI — is how the battery compounds instead of regressing. When our own engine had a soundness bug, we published it along with the fix and its regression tests.

Run it anywhere

CLI verifiedai audit <project> --all — sweep a project; JSON for machines, markdown for PR comments
GitHub Action drop one workflow file into any Lean repo — every PR gets its new statements audited, findings posted as a review comment
Self-hosted API verifiedai serve — zero-dependency HTTP server for on-prem use; proof corpora never leave your machines. Measured 1,278 theorems/min single-process, 160/160 seeded findings, zero false positives
MCP server agents that write proofs audit them before reporting success: audit_file and repair_project, structured JSON, works with Claude Code, Cursor, or any MCP client

Who it's for

Labs training provers. A model optimizing "the kernel accepted it" learns to smuggle axioms and mine contradictions — and its training data already contains both (see the Lean Workbook audit). verifiedai gates the reward signal and audits the corpus before the run, not in the postmortem.

Benchmark and dataset maintainers. Fixing miniF2F's statements moved measured prover accuracy from 40% to 70% (miniF2F-v2). A statement-integrity sweep is one command and about an hour per thousand statements, on a laptop.

Lean project maintainers. Every mathlib bump breaks downstream proofs; every PR can introduce a vacuous lemma reviewers wave through. Repair on bump, audit on PR — in CI.
