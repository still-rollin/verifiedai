verifiedai: Continuous Integration for Formal Proofs

The kernel checks your proofs. Nobody checks your statements.
Ayaan Siddiqui
BITS Pilani Hyderabad · INRIA/CNRS Paris · f20231060@hyderabad.bits-pilani.ac.in
July 2026 · Lean 4
A formally verified theorem is only as true as its statement. verifiedai is a statement auditor and proof-repair tool for Lean 4. Every audit finding is a compiled Lean probe — sound by construction, with no false positives — and every repair is accepted only if the kernel verifies it. Applied to four public corpora (2,744 statements), the auditor found 80 defective statements, including a vacuous item in the miniF2F benchmark and theorem … : n = n shipped twice in prover training data. Auditing costs approximately two compiles per file and runs unattended on commodity hardware.

1Results on public corpora

We run the auditor where the field's trust actually lives: the benchmarks provers are measured on, and the corpora they train on. Same battery, same claim discipline, raw JSON published.

Table 1: Statement audits of four public Lean 4 corpora (July 2026).

corpuswhat it isauditedfound
Lean Workbook[2] ~140k LLM-auto-formalized statements, used as prover training data 1,752 (seeded sample) 23 defective (1.31%) — incl. n = n twice, 1 vacuous; +9.8% no longer compiles audit
miniF2F-lean4 the benchmark every AI prover is measured on (LeanDojo port) 488 (all) 1 vacuous (hypothesis reads 1 = 1/5) + 50 trivial (10.2%) audit

The gradient is the point: maintained corpora are clean; static and auto-generated ones are not — and nothing in the ecosystem re-audits the static ones. A defective statement in a benchmark skews a score. The same statement in training data is a live reward signal teaching a prover to exploit contradictions. That is a CI problem, and CI is the product.

2What it does

verifiedai audit flags theorems the kernel happily accepts but that don't say what you think: contradictory (vacuous) hypotheses, statements that close under one tactic with all hypotheses stripped, unused hypotheses, and trust-chain cheats — smuggled axioms, hidden sorry, native_decide.

verifiedai repair fixes proofs broken by dependency bumps with a deterministic-first ladder — library search (exact?), tactic cascade, then an LLM with compile-error feedback. A candidate fix ships only if the file recompiles under the kernel. A wrong fix cannot ship, by construction.

$ verifiedai audit ./project Demo/Pricing.lean

  ✓ discount_le_price (line 10)
  ✗ big_discount_still_positive (line 16)
      [vacuous] the hypotheses are contradictory — False is derivable
      from them alone; this theorem proves nothing about any real case
  ⚠ tax_identity (line 22)
      [trivial] the conclusion holds without the hypotheses
      [unused_hypothesis] hypothesis `h` is never used

  3 theorems audited, 2 flagged

$ verifiedai repair ./project

  ✓ fixed Demo/Refunds.lean theorem refund_comm
      via exact? → exact Nat.add_comm a b
  1/1 broken declarations repaired (kernel-verified)

Figure 1: Audit and repair on the bundled demo project. Both commands are reproducible from the repository in under a minute.

3Method

The auditor turns each theorem into adversarial probe programs and lets the Lean kernel judge them. A probe that compiles is a mathematical fact about your statement, not an opinion.

3.1Parse the claim

Split every theorem into hypotheses and conclusion.

theorem big_discount_still_positive
  (p d : Nat) (h1 : d > p + 10)
  (h2 : d < p) :          -- hypotheses
  applyDiscount p d > 0   -- conclusion

3.2Attack it

The vacuity probe keeps the hypotheses and swaps the conclusion for False; the triviality probe keeps the conclusion and strips the hypotheses.

-- if this COMPILES, the theorem is about nothing
theorem __vac (p d : Nat)
  (h1 : d > p + 10) (h2 : d < p) : False := by
  first | omega | simp_all | decide

3.3Let the kernel judge

Probes are batched (at most 20 theorems per compile) and inserted inside each theorem's own namespace, so auditing costs about two compiles per file. Each probe carries a #print axioms sentinel: a finding requires positive confirmation, so if Lean ever aborts mid-file the engine fails safe rather than misreading silence.

4The checks

Table 2: The check battery. A live check that fires is a compiled probe — a proven fact.

checkwhat it catchesstatus
vacuous contradictory hypotheses — the theorem is true about nothing live
trivial conclusion closes under rfl / omega / decide / simp with all hypotheses stripped — no mathematical content survived formalization live
unused hypothesis assumptions the declaration never uses — over-constrained or mis-formalized live
trust chain the cheats an RL-trained prover learns: smuggled axioms, sorry hidden behind conjunctions or macros, native_decide — read from Lean's own axiom introspection. This is what makes verifiedai a reward-integrity gate for prover training live
faithfulness an LLM that never saw the informal statement back-translates the formal one; a separate judge compares (--faithful, BYO key). Soft signal — warnings only; calibration is an open problem[4] beta
definition drift redefine Prime locally, prove something easy about it — statements bound to shadowed definitions roadmap
spec mutation for verified code: inject known-buggy implementations; if the spec still verifies, the spec is too weak[3] roadmap
Rocq/Coq support the same battery for the Rocq ecosystem roadmap

Remark. False negatives are expected: the battery is a lower bound, and every check added widens it. The adversarial zoo — a corpus of kernel cheats the auditor must catch forever, enforced in CI — is how the battery compounds instead of regressing. When our own engine had a soundness bug, we published it along with the fix and its regression tests.

5Integration

CLI verifiedai audit <project> --all — sweep a project; JSON for machines, markdown for PR comments
GitHub Action drop one workflow file into any Lean repo — every PR gets its new statements audited, findings posted as a review comment
Self-hosted API verifiedai serve — zero-dependency HTTP server for on-prem use; proof corpora never leave your machines. Measured 1,278 theorems/min single-process, 160/160 seeded findings, zero false positives
MCP server agents that write proofs audit them before reporting success: audit_file and repair_project, structured JSON, works with Claude Code, Cursor, or any MCP client

6Who it's for

Labs training provers. A model optimizing "the kernel accepted it" learns to smuggle axioms and mine contradictions — and its training data already contains both (see the Lean Workbook audit). verifiedai gates the reward signal and audits the corpus before the run, not in the postmortem.

Benchmark and dataset maintainers. Fixing miniF2F's statements moved measured prover accuracy from 40% to 70%[1]. A statement-integrity sweep is one command and about an hour per thousand statements, on a laptop.

Lean project maintainers. Every mathlib bump breaks downstream proofs; every PR can introduce a vacuous lemma reviewers wave through. Repair on bump, audit on PR — in CI.
