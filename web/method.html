Method

Every finding is a compiled probe.

The auditor never renders an opinion. It writes small Lean programs that attack each theorem and lets the kernel judge them. A probe the kernel accepts is a proof.

∴The two core rules

For a theorem with hypotheses Γ and conclusion φ:

Γ ⊢ ⊥ (probe compiles)
vacuous(Γ ⊢ φ)
vac
∅ ⊢ φ (hypotheses stripped)
trivial(Γ ⊢ φ)
triv

Each premise is discharged by a Lean compilation, never a heuristic.

λPipeline

Parse

Split each theorem into hypotheses and conclusion.

theorem big_discount_still_positive
  (p d : Nat) (h1 : d > p + 10)
  (h2 : d < p) :          -- hypotheses
  applyDiscount p d > 0   -- conclusion

Probe

The vacuity probe keeps the hypotheses and swaps the conclusion for False. The triviality probe keeps the conclusion and strips the hypotheses.

-- if this compiles, the theorem is about nothing
theorem __vac (p d : Nat)
  (h1 : d > p + 10) (h2 : d < p) : False := by
  first | omega | simp_all | decide

Judge

Probes are batched, at most 20 theorems per compile, inside each theorem's own namespace. Cost: about two compiles per file. Every probe carries a #print axioms sentinel — a finding requires positive confirmation, so if Lean aborts mid-file the engine reports nothing rather than guessing.

✓The checks

checkwhat it catchesstatus
vacuous contradictory hypotheses. The theorem is true about nothing live
trivial the conclusion closes under rfl / omega / decide / simp with every hypothesis stripped live
unused hypothesis assumptions the declaration never uses live
trust chain smuggled axioms, sorry hidden behind conjunctions or macros, native_decide. Read from Lean's axiom introspection. This is the reward-integrity gate for prover training live
faithfulness blind back-translation of the formal statement, compared against the informal one by a separate judge. Warnings only beta
definition drift statements bound to shadowed or locally-redefined definitions roadmap
spec mutation break the implementation, keep the original proof, recompile — a surviving mutant means the spec never constrained that behavior. verifiedai mutate; first sweep: VERINA live
Rocq/Coq support the same battery for the Rocq ecosystem roadmap

Proposition (soundness). A finding that fires is a Lean probe the kernel accepted; no finding is a false positive. The converse fails: the battery is a lower bound. ∎

The adversarial zoo pins every cheat the auditor has ever caught as a CI contract. When the engine itself had a soundness bug, we published it with the fix and regression tests.

⇌The API

Everything is callable over HTTP. This transcript is verbatim: a local verifiedai serve auditing the ProofNet repository and repairing a broken proof.

$ verifiedai serve --port 8419
$ curl -s localhost:8419/v1/health
{"ok": true, "version": "0.5.0"}

$ curl -s localhost:8419/v1/audit -d '{
    "project_root": "experiments/Proofnet",
    "files": ["Proofnet/Axler.lean"], "ignore": ["sorry"]}'
{"reports": [{"file": "Proofnet/Axler.lean", "theorems": [
  {"name": "exercise_1_2", "findings": []},
  {"name": "exercise_1_3", "findings": [{"kind": "trivial", …}]},
  {"name": "exercise_1_4", "findings": [{"kind": "trivial", …}]},
  … 26 more theorems, clean …]}]}

$ curl -s localhost:8419/v1/repair -d '{"project_root": "demo"}'
{"ok": true, "repairs": [{"file": "Demo/Refunds.lean",
  "declaration": "theorem refund_comm (a b : Nat) : a + b = b + a",
  "fixed": true, "method": "exact? → exact Nat.add_comm a b"}]}
# 0.8 s round trip; the fix is applied to the file, kernel-verified

The server is stdlib-only Python. It runs inside your infrastructure; proof corpora never leave the machine.
