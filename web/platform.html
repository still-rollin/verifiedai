The larger system

An API layer for automated formal verification.

Proof assistants are the only technology that proves correctness instead of testing for the absence of failure. The platform's goal is to put them behind one API. The CI tool on the front page is its first shipped surface.

⇌The endpoints

POST /audit      → { project, files }            → findings          live today
POST /repair     → { project }                   → kernel-verified fixes   live today
POST /search     → { natural_language_query }    → ranked lemma matches    deployed (Rocqet)
POST /verify     → { spec, language }            → certificate | failure trace   in progress
POST /translate  → { rocq_proof, target: lean4 } → lean4 proof             research

⊕The systems

Each endpoint rests on a working system, built during formal-methods research at INRIA/CNRS Paris on proof-assistant interoperability.

systemlayerwhat it doesstatus
Rocqet retrieval semantic search over formal libraries. Natural-language queries against a dense vector index of the Rocq stdlib, MathComp, GeoCoq, UniMath and HoTT; no name recall needed deployed
Oraqle representation prover-agnostic proof extraction. Drives the Rocq kernel to capture, for every lemma application, the resolved arguments after unification — what the proof did, not what the surface syntax says. The layer behind /translate core validated; MCP live
ProofFilter search control neural-symbolic tactic filtering between the generator and the kernel: 11.8× solve-rate on miniF2F (1.7% → 20.1%), 76.7% of kernel calls pruned, 3.35× faster time-to-proof. What makes /verify viable at API scale under review, CICM 2026
verifiedai trust the statement auditor and repair bot. A proof certificate is only as good as the statement it certifies; this layer checks the statements shipped

A /verify call composes them: Rocqet retrieves lemmas, a tactic generator proposes steps, ProofFilter prunes them before the kernel, the auditor gates the result. Rocqet finds it, Oraqle explains it, verifiedai checks it.
